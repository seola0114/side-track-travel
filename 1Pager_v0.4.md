# 1Pager v0.4

## 문제 정의

### 1. 사용자와 페인포인트
- **사용자**: LLM으로 여행 일정을 짜고, 그 일정에 의존해 현지에서 움직이는 사람. 일정 대부분을 AI가 짜준 안에 맡기고, 현지에서는 이동 중이고 시간에 쫓기며 낯선 환경이라 긴 답변을 차분히 읽거나 일정을 직접 다시 짤 여유가 없다. 계획 의존도는 높은데 현장에서 직접 처리할 여유는 낮다.
- **페인포인트**: 여행지에서 변수(항공편 지연, 휴무, 길 막힘 등)가 생겼을 때, 바뀐 계획을 현장에서 실시간으로 반영하기가 어렵다.

### 2. 사용자의 목표 (Outcome)
- 일정에 변수가 생겼을 때, AI가 변경할 수 있는 부분은 자동으로 변경하고, 예약 변경 등 추가로 사용자의 행동이 필요한 항목을 쉽게 알고 편하게 재계획 할 수 있도록 한다.
- **완료 조건**: 변수를 입력한 뒤 몇 분 안에, 무엇이 그대로 유지되고 무엇을 바꿔야 하는지가 분간된 채 일정이 다시 확정된다.

### 3. 핵심 마찰
최종 결과물이 텍스트로만 존재해, 하나의 다룰 수 있는 결과물이 아니며 보기도 고치기도 어렵다.
- **보기**: 같은 대화창에서 추가 질문을 하면 최종 일정표가 위로 밀린다. 노션, PDF 등으로 따로 보관해야 한다.
- **반영**: 변경을 요청하면 정확히 어떤 부분이 변경됐는지 파악이 안 된 상태에서 직접 찾거나 여러번 반복 요청해야 한다.

## 해결책

### 4. 핵심 원칙
- **일정표는 대화 밖에 독립적으로 존재해야 한다** — 대화가 길어져도 일정표는 항상 제자리에서 볼 수 있어야 한다.
- **변수 대응은 비교로 해결한다** — 텍스트 재출력이 아니라, A/B안 비교로 영향 범위와 선택지를 한눈에 보여줘야 한다.
- **선택 한 번으로 확정되어야 한다** — 사용자가 공항 게이트에서 폰으로 조작하는 상황. 최소 탭으로 끝나야 한다.

### 5. 키스크린

**화면 구성**: 상단 일정표 (접기/펼치기) + 하단 대화

**핵심 순간**: 사용자가 "항공편 3시간 지연됐어"를 입력하면 →
1. 일정표가 자동으로 펼쳐지며 영향받는 셀이 시각적으로 표시됨 (빨간 dashed border + pulse 애니메이션)
2. AI가 영향 범위를 압축 요약하고 "A안 / B안 비교하기" 버튼을 제시 (pulse 애니메이션으로 주목 유도)
3. 일정 조정 모달이 열리면:
   - 시간별 A안/B안 비교 테이블이 나란히 표시됨
   - 선택한 안의 열은 선명하게, 비선택 열은 흐리게 표시
   - 추천 안(A안)에 추천 뱃지 + 추천 이유 표시
   - A안/B안 카드를 탭하여 선택 → "이 안으로 결정" 버튼으로 확정
4. 확정 시 일정표가 즉시 업데이트 (변경 셀 flash 애니메이션 + 자동 스크롤)
5. 마음이 바뀌면 "다른 안으로 변경" 버튼으로 재선택 가능

### 6. 인터랙션 설계
- **일정 초안 요청** → 바텀시트로 핵심 포인트 4가지 확인 → 맞춤 일정표 그리드 생성, 교통·통신 결정 제시
- **변수 입력** (예: "비행기 지연") → 일정표 자동 펼침, 영향받는 셀 시각 표시, A안·B안 비교 모달 제시
- **A/B안 선택** → 카드 탭으로 선택 상태 표시, 비교 테이블 열 연동, "이 안으로 결정"으로 확정
- **확정** → 일정표에 반영, 영향 없는 부분 유지, 변경 셀 flash 강조
- **재선택** → "다른 안으로 변경" 버튼으로 모달 재오픈, 이전 결정 취소 후 다시 선택
- **일정 항목 클릭** → 상세 모달에서 세부 정보 확인
- **메시지 전송 시** → 일정표 자동 접힘 (대화 공간 확보), 변수 입력 시 자동 펼침

### 7. 테스트 시나리오
1. 일정 초안을 요청한다 → 바텀시트 후 그리드가 생성되고 결정 항목이 제시되는 것을 확인한다.
2. 일정 항목을 클릭한다 → 상세 모달이 열리는 것을 확인한다.
3. "비행기 3시간 지연"이라고 입력한다 → 일정표가 펼쳐지고 영향받는 부분이 표시되고, A안·B안 비교 모달이 제시됨을 확인한다.
4. A안을 선택하고 "이 안으로 결정"을 누른다 → 일정표에 반영되고 나머지는 유지됨을 확인한다.
5. "다른 안으로 변경" 버튼을 누른다 → 모달이 다시 열리고 B안으로 재선택이 가능함을 확인한다.
