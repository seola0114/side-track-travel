# 1Pager v0.4

## 문제 정의

### 1. 사용자와 페인포인트
- **사용자**: LLM으로 여행 일정을 짜고, 그 일정에 의존해 현지에서 움직이는 사람. 일정 대부분을 AI가 짜준 안에 맡기고, 현지에서는 이동 중이고 시간에 쫓기며 낯선 환경이라 긴 답변을 차분히 읽거나 일정을 직접 다시 짤 여유가 없다. 계획 의존도는 높은데 현장에서 직접 처리할 여유는 낮다.
- **페인포인트**: 여행지에서 변수(항공편 지연, 휴무, 길 막힘 등)가 생겼을 때, 바뀐 계획을 현장에서 실시간으로 반영하기가 어렵다.

### 2. 사용자의 목표 (Outcome)
- 일정에 변수가 생겼을 때, AI가 변경할 수 있는 부분은 자동으로 변경하고, 예약 변경 등 추가로 사용자의 행동이 필요한 항목을 쉽게 알고 편하게 재계획 할 수 있도록 한다.
- **완료 조건**: 변수를 입력한 뒤 몇 분 안에, 무엇이 그대로 유지되고 무엇을 바꿔야 하는지가 분간된 채 일정이 다시 확정된다.

### 3. 핵심 마찰
최종 결과물이 텍스트로만 존재해, 하나의 다룰 수 있는 결과물이 아니며 보기도 고치기도 어렵다.
- **보기**: 같은 대화창에서 추가 질문을 하면 최종 일정표가 위로 밀린다. 노션, PDF 등으로 따로 보관해야 한다.
- **반영**: 변경을 요청하면 정확히 어떤 부분이 변경됐는지 파악이 안 된 상태에서 직접 찾거나 여러번 반복 요청해야 한다.

## 해결책

### 4. 핵심 원칙
- **일정표는 대화 밖에 독립적으로 존재해야 한다** — 대화가 길어져도 일정표는 항상 제자리에서 볼 수 있어야 한다.
- **변수 대응은 비교로 해결한다** — 텍스트 재출력이 아니라, A/B안 비교로 영향 범위와 선택지를 한눈에 보여줘야 한다.
- **선택 한 번으로 확정되어야 한다** — 사용자가 공항 게이트에서 폰으로 조작하는 상황. 최소 탭으로 끝나야 한다.

### 5. 키스크린
![키스크린](https://side-track-travel.vercel.app/tobe-mobile/)

**화면 구성**: 상단 일정표 (접기/펼치기) + 하단 대화

**핵심 순간**: 사용자가 "항공편 3시간 지연됐어"를 입력하면 →
1. 일정표가 자동으로 펼쳐지며 영향받는 셀이 시각적으로 표시됨 (빨간 dashed border + pulse)
2. AI가 영향 범위를 요약하고 "A안 / B안 비교하기" 버튼 제시
3. 모달에서 A안(카페 취소)과 B안(카페 늦추기)을 나란히 비교 — 추천 안에는 추천 뱃지 + 이유 표시
4. 한 번 탭으로 선택 → 일정표가 즉시 업데이트 (flash 애니메이션 + 자동 스크롤)
5. 마음이 바뀌면 "다른 안으로 변경" 버튼으로 재선택 가능

### 6. 인터랙션 설계
- **사용자가 대화에 일정 초안을 요청한다** → 일정표에 그리드가 생성되고, 결정이 필요한 항목(교통·통신)이 제시된다.
- **사용자가 대화에 변수를 입력한다** (예: "비행기 지연") → 일정표에서 영향받는 부분이 표시되고, A안·B안이 비교로 제시된다.
- **사용자가 A안·B안 중 하나를 선택한다** → 그 변경이 일정표에 반영되고, 영향 없는 부분은 그대로 유지된다.
- **사용자가 일정표의 한 항목을 클릭한다** → 상세 모달이 열려 그 일정의 세부를 그 자리에서 본다.

### 7. 테스트 시나리오
1. 일정 초안을 요청한다 → 그리드가 생성되고 결정 항목이 제시되는 것을 확인한다.
2. 일정 항목을 클릭한다 → 상세 모달이 열리는 것을 확인한다.
3. "비행기 3시간 지연"이라고 입력한다 → 일정표에 영향받는 부분이 표시되고, A안·B안이 비교로 제시됨을 확인한다.
4. 한 안을 선택한다 → 일정표에 반영되고 나머지는 유지됨을 확인한다.
5. "다른 안으로 변경" 버튼을 누른다 → 재선택이 가능함을 확인한다.

### 8. 프로토타입
- **링크**: https://side-track-travel.vercel.app/tobe-mobile/
- **키스크린 바로 보기**: 랜딩에서 "키스크린 바로 보기" 클릭 → 일정 확정 상태에서 바로 지연 시나리오 진입
- **전체 시나리오**: 랜딩에서 "전체 시나리오" 클릭 → 초안 요청 → 교통/통신 결정 → 지연 대응 풀 플로우
- **As-Is 비교**: https://side-track-travel.vercel.app/asis/ (같은 스토리를 일반 채팅으로 체험)
