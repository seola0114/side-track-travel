# 1Pager v0.4

## 문제 정의

### 1. 사용자와 페인포인트
- **사용자**: LLM으로 여행 일정을 짜고, 그 일정에 의존해 현지에서 움직이는 사람. 일정 대부분을 AI가 짜준 안에 맡기고, 현지에서는 이동 중이고 시간에 쫓기며 낯선 환경이라 긴 답변을 차분히 읽거나 일정을 직접 다시 짤 여유가 없다. 계획 의존도는 높은데 현장에서 직접 처리할 여유는 낮다.
- **페인포인트**: 여행지에서 변수(항공편 지연, 휴무, 길 막힘 등)가 생겼을 때, 바뀐 계획을 현장에서 실시간으로 반영하기가 어렵다.

### 2. 사용자의 목표 (Outcome)
- 일정에 변수가 생겼을 때, AI가 변경할 수 있는 부분은 자동으로 변경하고, 예약 변경 등 추가로 사용자의 행동이 필요한 항목을 쉽게 알고 편하게 재계획 할 수 있도록 한다.
- **완료 조건**: 변수를 입력한 뒤 몇 분 안에, 무엇이 그대로 유지되고 무엇을 바꿔야 하는지가 분간된 채 일정이 다시 확정된다.

### 3. 핵심 마찰
최종 결과물이 텍스트로만 존재해, 하나의 다룰 수 있는 결과물이 아니며 보기도 고치기도 어렵다.
- **보기**: 같은 대화창에서 추가 질문을 하면 최종 일정표가 위로 밀린다. 노션, PDF 등으로 따로 보관해야 한다.
- **반영**: 변경을 요청하면 정확히 어떤 부분이 변경됐는지 파악이 안 된 상태에서 직접 찾거나 여러번 반복 요청해야 한다.

## 해결책

### 4. 인터랙션 설계

화면은 **상단 일정표**(접기/펼치기)와 **하단 대화**로 구성된다. 일정표는 대화와 독립적으로 존재하며, 대화가 길어져도 항상 제자리에서 볼 수 있다.

**일정 생성**
- 사용자가 "도쿄 3박 4일 일정 짜줘"를 입력한다 → 바텀시트가 올라오며 핵심 포인트 4가지(여행 목적, 디즈니 여부, 항공편 시간, 숙소 위치)를 순서대로 묻는다 → 선택이 끝나면 바텀시트가 닫히고, 상단에 Day별 일정표 그리드가 생성된다. 대화에는 맞춤 일정 요약이 스트리밍으로 표시된다.

**결정 확정 (교통·통신)**
- AI가 "교통은 어떻게 할 생각이에요?"라고 묻는다 → 대화 입력창에 추천 답변("스이카가 나을듯")이 타이핑 애니메이션으로 미리 채워진다 → 사용자가 전송하면 일정표가 자동으로 접혀 대화 공간이 확보되고, AI 응답이 스트리밍으로 표시된다.

**변수 대응 (핵심 순간)**
- 사용자가 "항공편 3시간 지연됐어"를 입력한다 → 접혀있던 일정표가 자동으로 펼쳐지며, 영향받는 셀이 빨간 dashed border + pulse 애니메이션으로 표시된다. AI가 영향 범위를 6줄로 압축 요약하고, "A안 / B안 비교하기" 버튼이 pulse 애니메이션과 함께 나타난다.
- 사용자가 "A안 / B안 비교하기"를 탭한다 → 일정 조정 모달이 올라온다. 시간별 A안/B안 비교 테이블이 나란히 표시되며, 추천 안(A안)이 기본 선택 상태로 강조된다(2px accent border + 배경색). 비선택 안의 테이블 열은 흐리게(opacity 0.3) 처리되어 선택한 안에 시선이 집중된다. 추천 안에는 추천 뱃지와 "3시간 지연 후 바로 카페로 가면 체력 소모가 커요. 카페는 2~3일차에 다시 잡을 수 있어서 첫날은 가볍게 쉬는 게 나아요."라는 추천 이유가 표시된다.
- 사용자가 B안 카드를 탭한다 → B안이 selected 상태로 바뀌고, 테이블에서 A안 열이 흐려지고 B안 열이 선명해진다. A안 카드는 흐려진다.
- 사용자가 "이 안으로 결정" 버튼을 탭한다 → 모달이 닫히고, 일정표의 변경된 셀에 flash 애니메이션이 재생되며 해당 위치로 자동 스크롤된다. 대화에는 "B안으로 결정했어요"와 함께 다음 행동 안내("호텔에 늦는다고 메시지 보내기, 카페 예약 변경하기")가 스트리밍으로 표시된다.
- 마음이 바뀌면 "다른 안으로 변경" 버튼을 탭한다 → 이전 결정이 취소되고 모달이 다시 열려 재선택할 수 있다.

### 5. 테스트 시나리오

프로토타입 링크: https://side-track-travel.vercel.app/tobe-mobile/

**시나리오 A — 키스크린 직행 (변수 대응 핵심 순간)**

1. 랜딩에서 "키스크린 바로 보기"를 탭한다 → 일정이 확정된 상태의 대화 화면이 나타나고, 입력창에 "항공편 3시간 지연됐어"가 타이핑 애니메이션으로 채워지는 것을 확인한다.
2. 전송 버튼(↑)을 탭한다 → 일정표가 자동으로 펼쳐지며 Day 1의 셀들이 빨간색으로 변하는 것을 확인한다. 대화에 AI의 영향 요약과 "A안 / B안 비교하기" 버튼이 나타나는 것을 확인한다.
3. "A안 / B안 비교하기"를 탭한다 → 일정 조정 모달이 올라오고, A안(추천 뱃지 표시)이 기본 선택된 상태로 비교 테이블이 표시되는 것을 확인한다. B안 열이 흐리게 처리된 것을 확인한다.
4. B안 카드를 탭한다 → B안이 선택 상태로 바뀌고, 테이블의 A안 열이 흐려지고 B안 열이 선명해지는 것을 확인한다.
5. "이 안으로 결정"을 탭한다 → 모달이 닫히고, 일정표에서 변경된 셀이 깜빡이며 강조되는 것을 확인한다. 대화에 결과 메시지가 스트리밍으로 표시되는 것을 확인한다.
6. "다른 안으로 변경"을 탭한다 → 모달이 다시 열리고 A안으로 재선택할 수 있는 것을 확인한다.

**시나리오 B — 전체 플로우**

1. 랜딩에서 "전체 시나리오"를 탭한다 → 대화 화면이 열리고 바텀시트가 올라오는 것을 확인한다.
2. 4가지 질문에 순서대로 답한다 → 바텀시트가 닫히고 상단에 Day별 일정표가 생성되는 것을 확인한다.
3. 입력창에 채워진 텍스트("스이카가 나을듯")를 전송한다 → 일정표가 접히고 AI가 교통 결정을 확인하는 것을 확인한다.
4. 통신 답변("각자 eSIM으로")을 전송한다 → AI가 확정하고 "변수가 생기면 알려주세요"라고 안내하는 것을 확인한다.
5. "항공편 3시간 지연됐어"를 전송한다 → 시나리오 A의 2~6번과 동일하게 변수 대응 흐름을 체험한다.

**As-Is 비교**: https://side-track-travel.vercel.app/asis/ 에서 같은 스토리를 일반 채팅으로 체험하면, 텍스트로만 관리되는 일정의 불편함을 직접 느낄 수 있다.
